apply调用模式

以上四种常见类型比较容易判断this的指向，但是当this与闭包、立即执行函数、原型链、作用域链等概念同时出现时，this的指向就变的难以判断了。在分析混合出现情况之前，我们先澄清几个知识点。
1.全局环境中变量的声明

上述例子中的b没有通过var声明，所以它实际上不是一个变量，而是global（window）的一个property。变量和属性的显著区别就是，变量不可以被delete而属性可以。

2.this的取值，由函数的调用方式决定而不是函数的定义

一个错误的观点是，如果函数被定义成全局函数，this指向global，如果被定义成对象的方法，则指向对象。

上述例子中，baz与foo.bar的定义是等价的，但是最终的this指向却不同，所以this的取值不是由函数定义决定的，而是由调用者。

this X new

两种情况返回
test(): 5,10，10，5
new test():5,undefined,undefined,5
第一个很好理解，由于变量声明的提升，a=5变成了局部变量，this.a指向全局变量。
第二个new test()，new之后this绑定到新生成的对象上，在对象里调用this，也就是在对象的方法里调用this，此时this指向的是包含方法的对象，也就是test{}，而这里我们并没有声明test对象中的a属性，所以返回值是undefined。
