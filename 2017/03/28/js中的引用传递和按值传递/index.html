Javascript中按值传递&按引用传递从看懂到看开

Envy
2017-03-28
Javascript


Javascritpt高级程序设计第三版，p69-71详细介绍了，两种类型传递以及函数参数传递的原理。书中提到， js中所有函数的参数都是按值传递的 也就是说，把函数外部的值复制给函数内部的参数，就和把值从一个变量复制到另一个变量一样。而变量的传递有按值和按引用两种，但参数只能按值传递。想给参数传递基本类型就是把值复制给参数。但是传递引用类型的值时，会把这个值在内存中的地址复制给一个局部变量，因此这个局部变量的变化会反映在函数的外部。
此外书中还给出了例子：

作者认为如果person是按引用传递的，那么person就会自动被修改为指向其name属性值为”Greg”的新对象。但是，当接下来再访问person.name时，显示的值仍然是”Nicholas”。这说明即使在函数内部修改了参数的值，但原始的引用仍然保持不变。实际上，当在函数内部重写obj时，这个变量引用的就是一个局部对象了。而这个局部对象会在函数执行完毕后立即被销毁。
从这个例子作者认为，函数的参数传递是按值传递的。
但是他说的真的对吗？

我们换一种写法，作者说“如果person是按引用传递的,那么person就会自动被修改为指向其name属性值为Greg的新对象”
