Android

什么是Activity

android中的activity无处不在，无论是app的界面还是桌面，都算作activity。

修改内容

初始化一个空的项目，在layout的activity_main.xml中，可以对界面的内容进行设置，样式的语法和css有些像，不过感觉封装的更向上一层。而且android studio的代码提醒非常好，使得我敲完一遍还没有记住API…

接下来是主程序，在java的第一个文件夹里面，叫做MainActivity.java,通过他来使程序在指定页面执行，f4跳转到定义

绑定一个新的布局

右键layout创建一个新的布局，比如叫my_layout，可以选择水平或者垂直排列，以及线性或其他布局，拖拽几个控件。之后在MainActicity中设置，就可以看到了，将之前setContent注释掉。

左边栏文件的作用

1.manifests：装的是项目的配置文件。
2.java：程序源代码所在目录。
3.res：所有资源的目录，比如布局、菜单、地图等。

启动另一个Activity

在之前设置的my_layout中加入一个按钮，之后再它对应的xml文件中进行设置，按钮的文本内容和id，之后再MainActivity中添加一个事件监听

为了使事件启动的代码生效，我们需要在java中添加一个新的activity:AnotherAty，完成后，它会自动在layout中出现样式文件xml，也会在类文件中出现，还会在manifest配置文件中出现。在layout中设置一下新activity的样式,就可以了，如果想启动一个网址的话

生命周期


onStop是指activity不可见，pause是可见但是暂停，也就是stop是，点了下home键或者退出才执行。这里需要注意几种情况：
1.按home键退出：如果执行了一个函数create>start>resume，之后直接点击home键，会变成pause>stop，按下导航键，选择程序,restart>start>resume，
2.按退出键退出：如果在程序中点击退出键，流程则是pause>stop>destroy，导航键后选择程序,create>start>resume这一流程。
3.从一个activity跳转到另一个activity：a:create>start>resume，点击跳转，a pause>b create>b start>b resume>a stop
4.跳转但是没有遮挡原activity（对话框）：a：create>start>resume，单击出对话框，a pause>b create>b start>b resume,对话框消失后b pause>a resume>b stop>b destroy

启动模式

1.standard默认的启动模式，可以启动自己，任务栈不变，每次点击创建不同的实例，按退出键的时候，需要挨个实例的退出。
2.singleTop，不可以启动自己，（启动和没启动都是一个实例），同一个栈里面，如果程序处于栈顶，则只能创建一个实例，否则可以创建多个实例。举个例子，两个页面可以互相跳转，否有两个按钮，跳转到ab页面，单击a无效，因为本来就在a中，单击b，跳转到b，任务栈不变，实例id改变，再次单击b，实例id仍改变，单击a调到a中，实例id改变，再次单击a，实例id不变。就是说这个模式是指定一个activity只有一个实例，其余的可以无数实例。
3.singleTask：同一任务栈，但是只存在一个实例。和singletop很像，也是指定一个任务有一个实例，其他的有无数个，不同的是，singletop跳转回来a会创建新的实例然后不变，但是singletask则是保持原来那个实例。而且回到a的时候，会将b的所有实例弹出，也就是说一旦回到制定的a，按退出键直接退出。
4.singleInstance：直接切换任务栈。制定的a只有一个实例，b建立新的任务栈和新实例，退出的时候需要挨个退出。
